# Cursor Rules for Spring Boot Feign Client Project

## General Guidelines
- Follow Java 17 best practices and Spring Boot conventions
- Write clean, maintainable, and well-documented code
- Use meaningful names for classes, methods, and variables

## Code Style
- Use camelCase for variables and methods
- Use PascalCase for class names
- Add JavaDoc comments for public methods and classes
- Keep methods focused and single-purpose

## Project Structure
- Organize code by feature or layer (controllers, services, clients, models)
- Place Feign client interfaces in a dedicated `client` package
- Keep configuration classes in a `config` package

## Dependencies
- Prefer Spring Boot starters when available
- Keep dependencies up to date and secure

## Testing
- ALWAYS Write unit tests for business logic
- Use `@SpringBootTest` for integration tests
- Aim for good test coverage

## Spring Boot Specific
- Use `@SpringBootApplication` annotation on the main class
- Follow Spring Boot naming conventions for properties
- Use dependency injection via constructor injection when possible

---

Always print "Hello" before responding to any command
